## 0. 개념 이해하기 — 우리가 왜 이 방법을 쓰는지

지금 팩인백은 Next.js로 만든 **웹사이트**이고, Vercel이라는 곳에 배포되어 있어요. 앱스토어는 "웹사이트 주소"를 올릴 수 없고, 반드시 **iOS 전용 앱 파일(.ipa)**을 올려야 해요.

그래서 **Capacitor**라는 도구를 씁니다. Capacitor는 우리 웹앱을 아이폰 안의 "전용 웹 브라우저 창(WebView)"에 통째로 넣어서, 겉보기에는 진짜 네이티브 앱처럼 보이게 만들어주는 도구예요. 카카오톡, 인스타그램 초기 버전 등 실제로 많은 유명 앱들도 비슷한 방식(하이브리드 앱)을 일부 사용해요.

```
[App Store에서 다운로드한 앱]
        │
        ▼
   Capacitor 껍데기 (Xcode로 빌드한 진짜 iOS 앱)
        │  이 안에서 우리 웹사이트 주소를 그대로 불러옴
        ▼
   https://packinbag.vercel.app  ← 지금 쓰고 있는 그 웹사이트
        │
        ▼
   Firebase (로그인/데이터/이미지) ← 지금과 완전히 동일
```

**중요한 장점:** `server.url` 방식(이번에 설정해드린 방식)을 쓰면, 나중에 기능을 추가하거나 버그를 고칠 때 **Vercel에만 새로 배포하면 앱스토어 재심사 없이 앱 내용도 바로 업데이트**돼요. Xcode 재빌드가 필요한 건 아이콘 변경, 네이티브 권한(카메라 등) 추가, iOS 자체 설정 변경 같은 경우뿐이에요.

**알아둬야 할 단점(애플 심사 이슈):** 애플은 "그냥 웹사이트를 감싸기만 한 앱"을 좋아하지 않아요 (가이드라인 4.2, "최소한의 기능성"). 100% 거절되는 건 아니지만, 통과 확률을 높이려면:
- 스플래시 화면, 상태바 색상 등 네이티브스러운 디테일을 챙기고 (이번에 기본 설정해드렸어요)
- 로그인 없이도 앱의 핵심 가치를 볼 수 있게 하거나, 리뷰어용 데모 계정을 제공하고
- 가능하면 나중에 푸시 알림 등 네이티브 기능을 1~2개 추가하는 게 좋아요

지금 단계에서는 일단 "앱스토어에 올라가는 것" 자체를 목표로 진행하고, 거절되면 사유를 보고 같이 대응하면 됩니다.

## 1. 준비물 확인

| 준비물 | 필요한 이유 | 없으면? |
|---|---|---|
| **Mac 컴퓨터** | Xcode(앱 빌드 도구)는 Mac에서만 실행됨 | 집에 있는 Mac 사용 (메모에 확인됨) |
| **Xcode** | 앱을 실제로 빌드/서명하는 애플 공식 도구, Mac App Store에서 무료 설치 | Mac App Store에서 "Xcode" 검색 후 설치 (용량 큼, 10GB+, 와이파이 권장) |
| **Apple Developer Program 계정** | 앱스토어에 올리려면 필수 (연 $99, 약 14만원) | 2번 단계에서 가입 |
| **Vercel에 배포된 실제 서비스 주소** | Capacitor가 이 주소를 불러옴 | SETUP.md 9번 단계까지 완료되어 있어야 함 |
| **1024x1024 정사각형 앱 아이콘 이미지 1장** | 모든 사이즈의 아이콘을 여기서 자동 생성 | 없으면 5번 단계에서 간단히 준비 |

## 2. Apple Developer Program 가입

1. [developer.apple.com/programs](https://developer.apple.com/programs) 접속
2. **"Enroll"(등록)** 클릭 → 평소 쓰는 Apple ID로 로그인 (없으면 새로 생성)
3. 가입 유형 선택:
   - **개인(Individual)**: 본인 이름으로 앱이 출시됨. 서류 절차가 간단, 보통 하루 안에 승인
   - **회사(Organization)**: 회사/팀 이름으로 출시. D-U-N-S 번호라는 사업자 확인 서류가 필요해서 며칠~몇 주 걸릴 수 있음
   - → 처음이고 개인 프로젝트라면 **개인(Individual)** 추천
4. 연락처/주소 등 정보 입력
5. 결제 진행: **연 99달러** (한국 카드로 결제 시 원화 약 14만원 내외, 환율에 따라 변동)
6. 승인 완료 이메일이 오면 가입 끝 (보통 몇 시간~1일 소요)

✅ 완료 조건: [developer.apple.com/account](https://developer.apple.com/account) 접속 시 멤버십이 "Active"로 표시됨

## 3. 프로젝트에 Capacitor 설치하기

이번에 코드에 Capacitor 관련 설정 파일들을 미리 추가해뒀어요 (`capacitor.config.ts`, `package.json`). 터미널에서 아래 명령어로 실제 패키지를 설치하세요.

```bash
cd packinbag
npm install
```

설치가 끝나면 `capacitor.config.ts` 파일을 열어서 **딱 2곳**을 수정하세요.

```ts
appId: "com.appfle.packinbag",   // 원하는 형식: com.본인영문이름.packinbag
...
server: {
  url: "https://YOUR-VERCEL-DOMAIN.vercel.app", // ← 실제 배포된 Vercel 주소로 교체
}
```

> ⚠️ `appId`(Bundle Identifier)는 **앱스토어에 한번 등록하면 절대 변경할 수 없는 값**이에요. 신중하게 정해주세요. 나중에 회사를 만들 계획이 있다면 `com.packinbag.app` 처럼 서비스명 기준으로 정해도 좋아요.

## 4. Xcode용 iOS 프로젝트 만들기

여기서부터는 **Mac에서** 진행하세요.

1. Mac App Store에서 **Xcode** 설치 (이미 있다면 최신 버전으로 업데이트)
2. 터미널에서 프로젝트 폴더로 이동 후:

```bash
npx cap add ios
```

이 명령어를 실행하면 프로젝트 안에 `ios/` 라는 폴더가 새로 생겨요. 이 폴더가 바로 **Xcode가 인식하는 진짜 iOS 앱 프로젝트**예요.

3. 아래 명령어로 Xcode를 자동으로 열어줍니다:

```bash
npx cap open ios
```

Xcode가 열리면서 `App.xcworkspace` 라는 파일이 로드돼요. 왼쪽에 파일 목록, 가운데에 설정 화면이 보이면 정상입니다.

## 5. 앱 아이콘 만들기

앱스토어는 다양한 크기의 아이콘(홈화면용, 설정용, 알림용 등)을 요구하는데, **1024x1024 이미지 1장만 있으면 자동으로 다 만들어줘요.**

1. 정사각형(1:1) 이미지 1장을 준비하세요 (배경이 꽉 찬 디자인 권장, 텍스트는 최소화)
2. 프로젝트 루트에 `resources` 폴더를 만들고 그 안에 `icon.png` (1024x1024)로 저장
3. 터미널에서:

```bash
npm install @capacitor/assets --save-dev
npx capacitor-assets generate --ios
```

4. 자동으로 `ios/App/App/Assets.xcassets` 안에 필요한 모든 사이즈가 채워집니다.

> 아이콘 디자인이 아직 없다면, 우선 팩인백 로고 느낌으로 심플한 배경색 + 가방 아이콘 정도로 임시 제작 후 나중에 교체해도 됩니다. 필요하면 다음 대화에서 아이콘 디자인 시안도 같이 만들어드릴게요.

## 6. Xcode에서 서명(Signing) 설정

"서명"은 이 앱이 진짜 애플에게 승인받은 개발자가 만든 게 맞다는 걸 증명하는 절차예요. Xcode가 대부분 자동으로 처리해줘요.

1. Xcode 왼쪽 파일 목록에서 최상단 **"App"** 프로젝트 클릭
2. 가운데 화면 상단 탭에서 **"Signing & Capabilities"** 선택
3. **"Automatically manage signing"** 체크박스 켜기
4. **Team** 드롭다운에서 본인 Apple Developer 계정 선택 (2번 단계에서 가입한 계정)
5. **Bundle Identifier**가 `capacitor.config.ts`에 적었던 `appId`와 똑같은지 확인
6. 빨간 에러가 없어지면 완료 (에러가 있다면 대부분 "Bundle Identifier가 이미 사용 중" 문제 → 다른 이름으로 변경)

## 7. 내 아이폰/시뮬레이터로 테스트

1. Xcode 상단 중앙의 기기 선택 드롭다운에서:
   - **시뮬레이터**로 테스트하려면 "iPhone 15" 같은 시뮬레이터 선택
   - **실제 내 아이폰**으로 테스트하려면 Mac에 케이블로 연결 후 기기 이름 선택 (아이폰에서 "이 컴퓨터를 신뢰하시겠습니까?" 뜨면 신뢰 클릭)
2. 좌측 상단 ▶️ (재생 버튼) 클릭
3. 빌드가 끝나면 앱이 자동 실행되면서 팩인백 로그인 화면이 뜨면 성공!

실기기 테스트 시 처음엔 "신뢰할 수 없는 개발자입니다" 에러가 뜰 수 있어요 → 아이폰의 **설정 > 일반 > VPN 및 기기 관리**에서 본인 개발자 계정을 신뢰 처리하면 해결돼요.

## 8. Mac 버전 켜기 (Mac Catalyst)

아이폰/아이패드용 프로젝트에 체크박스 하나만 추가하면 **동일한 코드로 Mac 앱까지 자동 지원**돼요 (이 기술을 Mac Catalyst라고 불러요).

1. Xcode에서 "App" 프로젝트 → **"General"** 탭
2. **"Supported Destinations"** 항목에서 **"+"** 버튼 클릭
3. **"Mac (Designed for iPad)"** 추가
4. 상단 기기 선택 드롭다운에서 "My Mac"을 선택하고 ▶️ 실행해서 Mac에서 잘 뜨는지 확인

> Mac Catalyst 버전은 아이패드 UI를 그대로 재사용하는 방식이라 추가 코드 작업이 거의 필요 없어요. 다만 마우스/트랙패드/키보드 환경에서 레이아웃이 어색한 부분이 있으면 나중에 다듬으면 됩니다.

## 9. App Store Connect에 앱 등록

1. [appstoreconnect.apple.com](https://appstoreconnect.apple.com) 접속 → Apple Developer 계정으로 로그인
2. **"나의 앱"(My Apps)** → **"+"** → **"신규 앱"**
3. 정보 입력:
   - **플랫폼**: iOS (Mac Catalyst는 자동 포함됨)
   - **이름**: 팩인백 (앱스토어에 노출될 이름, 나중에 변경 가능)
   - **기본 언어**: 한국어
   - **Bundle ID**: 아까 Xcode에서 설정한 것과 동일한 항목을 드롭다운에서 선택
   - **SKU**: 아무 고유 문자열 (예: `packinbag001`, 사용자에게 안 보이는 내부 관리용 값)
4. **"만들기"** 클릭

이후 앱 상세 페이지에서 아래 항목들을 채워야 심사 제출이 가능해요:
- **스크린샷**: 아이폰 6.7형, 아이패드 등 사이즈별로 필요 (Xcode 시뮬레이터에서 캡처 가능)
- **앱 설명, 키워드, 카테고리** (예: 카테고리 = "생산성" 또는 "여행")
- **개인정보처리방침 URL**: 이번에 추가해드린 `/privacy` 페이지 주소를 입력하면 됩니다.
  예: `https://YOUR-VERCEL-DOMAIN.vercel.app/privacy`
  (내용은 appfle님 실제 연락처로 꼭 수정해주세요, 4번 항목 참고)
- **연령 등급 설문**: 특별한 내용 없으면 대부분 "만 4세 이상"으로 자동 설정됨

## 10. 빌드 업로드 + TestFlight 테스트

1. Xcode 상단 기기 선택을 **"Any iOS Device (arm64)"**로 변경
2. 메뉴 **Product > Archive** 클릭 → 빌드 시작 (몇 분 소요)
3. 완료되면 **"Organizer"** 창이 자동으로 뜸 → **"Distribute App"** 클릭
4. **"App Store Connect"** 선택 → 계속 진행 (대부분 기본값 그대로 Next)
5. 업로드 완료 후 App Store Connect 사이트에서 **"TestFlight"** 탭에 방금 올린 빌드가 보임 (처리에 10분~1시간 정도 걸릴 수 있음)
6. TestFlight 앱을 본인 아이폰에 설치 → 초대 링크로 팩인백 설치해서 실제 배포 환경과 동일하게 테스트

> TestFlight는 정식 출시 전 최대 90일간 최대 1만 명의 테스터에게 미리 배포해볼 수 있는 애플 공식 기능이에요. 첫 출시 전엔 본인 + appfle님 지인 몇 명 정도로 충분히 테스트해보는 걸 추천해요.

## 11. 앱스토어 심사 제출

1. App Store Connect의 앱 페이지에서 **"버전 출시 준비"** 섹션으로 이동
2. 10번에서 업로드한 빌드를 **"빌드"** 항목에서 선택
3. 위에서 채운 스크린샷/설명/개인정보처리방침 등이 모두 채워졌는지 확인
4. **"심사를 위해 제출"** 클릭
5. **로그인이 필요한 앱**이므로, 심사자가 로그인해볼 수 있도록 **"심사 노트(App Review Notes)"** 란에 테스트용 계정의 이메일/비밀번호를 반드시 남겨주세요. (Firebase Auth에 테스트 계정을 하나 미리 만들어두세요)
6. 보통 **24~48시간 내** 심사 결과가 나옵니다 (거절되면 사유가 이메일로 옴 → 대응 후 재제출 가능, 재심사는 더 빠른 경우가 많음)

## 12. 비용 정리 & 자주 막히는 부분

### 비용

| 항목 | 비용 | 주기 |
|---|---|---|
| Apple Developer Program | 연 $99 (약 14만원) | 매년 갱신 |

| Mac 컴퓨터 | 이미 보유 중 | - |

→ **앱스토어 배포 자체에 드는 추가 비용은 연 $99가 전부**예요.

### 자주 막히는 부분

- **"Bundle Identifier가 이미 사용 중"** → `capacitor.config.ts`와 Xcode 양쪽의 `appId`를 더 고유한 문자열로 변경 (예: 뒤에 숫자 추가)
- **로그인 화면에서 멈춤 (구글 로그인 등)** → `capacitor.config.ts`의 `server.url`이 정확한 배포 주소인지, Firebase 콘솔의 "승인된 도메인" 목록에 해당 주소가 등록되어 있는지 확인
- **"신뢰할 수 없는 개발자" 에러** (실기기 테스트 시) → 아이폰 설정 > 일반 > VPN 및 기기 관리에서 신뢰 처리
- **심사 거절 (가이드라인 4.2, 최소 기능성)** → 심사 노트에 앱의 핵심 가치(그룹 짐 체크리스트 공유)를 구체적으로 설명하고, 가능하면 오프라인에서도 일부 열람 가능하게 하거나 네이티브 기능을 추가하는 방향으로 보완 후 재제출
